## JavaScript

Then moving forward, I jumped into JavaScript. JavaScript is the most used language for developing web applications. So, I started with javascript. It was quite a hype of JS back then among the beginners, but trust me it's a very good and easy Language. 

Now, Javascript has two flavors on its own. You can use javascript to manipulate DOM or you can simply use javascript to write some programs and do nothing. Beginners often get confused about this situation. So, Let me clear up all doubts. 

In any programming language, you will have basics such as declaring variables, loops, conditionals, etc. When you just write these in JavaScript, it is called Vanilla JavaScript. But, JavaScript can also be used to manipulate the website's DOM ( Document Object Model ). As a developer, you have to know both of these. But, you should be clear about what you are learning and why you are learning. 

I would suggest you go through this Codecademy Course - Link This will teach you only Vanilla JavaScript. To use JavaScript in your websites to add interactivity, you need to know the basics.

Then, You can Learn DOM manipulations. A very good interactive course is on Khan Academy, which I personally used back then. Here is the Link

## React & Next

After I was done with JavaScript, I felt confused about what frameworks to learn. I had options for React, Angular, and Vue. But, the MERN stack was on Hype at that time ( which it still is in 2023 ). So, I decided to go with React Library. I watched the playlist of Net Ninja. Here is the [Playlist](https://www.youtube.com/watch?v=j942wKiXFu8&list=PL4cUxeGkcC9gZD-Tvwfod2gaISzfRiP9d). 

The Playlist will teach you the basic of React JS and hooks. and some API fetching. And not to mention you will have to choose between Class-based or Function-based when you start with React JS. Function-based is the latest, So I would prefer to go with Functional Development along with some knowledge of the Component Life Cycle which is followed in Class-Based Development

Then at this point, You must be frustrated with making the same UI all the time, so You might want to learn a component library like the Material UI. You can use this playlist to learn MUI.

After Learning some of React and MUI, you can make a lot of stuff. Not to mention you can use Vercel to deploy all your React Projects.

Trust me after this, things became easier. Making UI got a little faster and less frustrating.
